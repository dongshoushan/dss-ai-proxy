# 人工手动构建说明

本文档说明如何手动进行项目打包、编译、Docker镜像制作及启动。

---

## 环境要求

- **JDK版本**：21
- **构建工具**：Maven 3.6+
- **容器工具**：Docker

---

## 一、项目打包与编译

### 1.1 Maven 打包

在项目根目录执行以下命令：

```bash
# 清理并打包（跳过测试）
mvn clean package -DskipTests

# 或者执行完整构建（包含测试）
mvn clean package
```

打包完成后，JAR包位于 `target/` 目录下，文件名为 `dss-ai-proxy-0.0.1-SNAPSHOT.jar`。

### 1.2 编译验证

```bash
# 仅编译，不打包
mvn compile
```

---

## 二、Docker 镜像制作

### 2.1 构建镜像

在项目根目录执行：

```bash
# 基础构建命令
docker build -t dss-ai-proxy:latest .

# 带版本号构建
docker build -t dss-ai-proxy:1.0.0 .
```

### 2.2 查看构建结果

```bash
# 查看本地镜像
docker images | grep dss-ai-proxy
```

---

## 三、Docker 启动命令

### 3.1 带挂载路径启动

```bash
docker run -d \
  --name dss-ai-proxy \
  --restart=always \
  -p 30089:30089 \
  -v /home/dss-ai-proxy/config:/app/config \
  dss-ai-proxy:latest
```

### 3.2 挂载路径说明

| 宿主机路径 | 容器路径 | 说明 |
|-----------|---------|------|
| `/home/dss-ai-proxy/config` | `/app/config` | 外部配置文件目录 |

### 3.3 端口说明

| 宿主机端口 | 容器端口 | 说明 |
|-----------|---------|------|
| `30089` | `30089` | 应用服务端口 |

---

## 四、常用运维命令

### 4.1 容器管理

```bash
# 查看容器状态
docker ps -a | grep dss-ai-proxy

# 停止容器
docker stop dss-ai-proxy

# 启动容器
docker start dss-ai-proxy

# 重启容器
docker restart dss-ai-proxy

# 删除容器
docker rm -f dss-ai-proxy
```

### 4.2 日志查看

```bash
# 查看实时日志
docker logs -f dss-ai-proxy

# 查看最近100行日志
docker logs --tail 100 dss-ai-proxy

# 查看指定时间段日志
docker logs --since "2024-01-01" dss-ai-proxy
```

### 4.3 进入容器

```bash
# 进入容器内部
docker exec -it dss-ai-proxy /bin/bash
```

### 4.4 镜像管理

```bash
# 删除镜像
docker rmi dss-ai-proxy:latest

# 导出镜像
docker save -o dss-ai-proxy.tar dss-ai-proxy:latest

# 导入镜像
docker load -i dss-ai-proxy.tar
```

---

## 五、注意事项

1. **端口冲突**：确保宿主机 30089 端口未被占用
2. **目录权限**：挂载目录需确保有正确的读写权限
3. **配置文件**：首次部署需在挂载目录创建 `application.yml` 配置文件
4. **资源限制**：生产环境建议配置内存和CPU限制
5. **日志轮转**：建议配置日志轮转策略避免磁盘占满
